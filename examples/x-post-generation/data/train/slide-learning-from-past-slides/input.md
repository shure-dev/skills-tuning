# テーマ
スライド生成Skillsを過去のスライドから自力で学習させる実験の改善版

# やったこと / 素材
- テキストの骨子を渡して過去に作ったスライドと同じテイストで自動生成するのが目標
- Claude CodeのSkills（AIへの指示書）を自動で最適化している
- 仕組みは機械学習と同じループ：①骨子からスライド生成 → ②お手本と比較して採点 → ③何がダメだったか分析 → ④指示書(Skills)を書き換え、を繰り返す
- 前回の実験（引用元投稿あり）では2回で改善が頭打ちだった
- 前回の問題と今回の改善：
  1. 学習データの質が低かった → 本質的に学習に貢献できる質のいいスライドのみを使用し、データ量を7枚→18枚に増量
  2. 採点があいまいだった → 4軸（テキスト/構造/レイアウト/スタイル）でお手本と厳密比較するようにした
  3. Skillsの更新が小さすぎた → 毎回のイテレーションでSkillsを大幅に更新するようにオプティマイザーを改善

# 結果 / データ
- スコア 59.7 → 91.6 に改善
- 前回と違い3回目まで伸び続けた
- 4回目は87.3に下がった（Skillsでも「学習しすぎ」が起きる）
- 学習・テストデータに含まれない未知の入力でもスライド生成の品質が向上（スキルなし/学習1回目/ベストの比較画像あり）

# 考察 / 気づき
- データセットを作るのが難しいという課題がある
- 過去に人間が作ったスライドはあっても、そのスライドを作るのに使ったリサーチ結果や議事録などのインプットデータを正確に見つけてくるのが難しい

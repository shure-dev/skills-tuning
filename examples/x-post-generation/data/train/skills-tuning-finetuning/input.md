# テーマ
Skillsのファインチューニングで生成物の細かな品質が改善するという発見

# やったこと / 素材
- 過去の生成物を用いてスライド生成Skillsを自動最適化する実験
- 2つの手法を比較：
  - 手法1（ベーススキル生成）：Claude Codeに過去の成果物（実務で作ったスライド）を読ませるだけでSkillsを作らせる（画像1枚目の手法）
  - 手法2（ファインチューニング）：実際にSkillsを実行してお手本と比較し、何がダメだったか分析してSkillsを書き直すフィードバックループ（画像2枚目の手法）
- ディープラーニングのファインチューニングとのアナロジー：事前学習で70%まで行くが、そこから先はタスク固有のデータで反復的に最適化が必要
- 15パターンのダミーデータで「スキルなし」「ベーススキル」「チューニング後スキル」の3条件を比較（画像3枚目）
- 青の枠（ベース）と赤の枠（チューニング後）でレイアウトの綺麗さを比較

# 結果 / データ
- ベーススキル生成だけでだいたい70%くらいの品質が出る
- 70%→80%、80%→85%に持っていくにはフィードバックループが必要
- 見えたこと3点：
  1. タイトルスライドのようなシンプルなケースではスキルなし/ベース/チューニング後でほぼ差がない
  2. 差が出るのは構造的な複雑なケース：2カラム、テーブル、フロー図、フッターバー付きスライドなど「どう構造を組むか」の判断が必要なケース
  3. チューニングで学習されたのは「レイアウトの構造判断」かもしれない。対比関係なら2カラム、テーブルデータならグリッド、フロー表現なら矢印付き横並びなどの判断ルールがスキルに蓄積

# 考察 / 気づき
- チューニングによって「細かな箇所」が学習されてSkillsに蓄積される
- 過去の成果物に眠っている暗黙知（レイアウトの組み方、色の使い方、要素の分割ルール）をAIが読める形式のスキルとして構造化して保存できる
- 「事前学習（ベーススキル生成）+ ファインチューニング（フィードバックループ）」がパーソナルで品質の良いAIエージェントを育てる本質的な方向性
- まだ確立された手法ではなくサンプル数も少ないので断言はできない
- 汎用的なAIが「自分の仕事のやり方」を緻密に少しずつ学んでいく

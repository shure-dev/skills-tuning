# テーマ
Skillsを学習させるためのデータセットを成果物から逆算して自動生成する手法

# やったこと / 素材
- Skillsの自動最適化にはデータセットが必要
- 過去に人間が生成したスライドなどの成果物はたくさんあるが、それを作るのに使ったリサーチ結果や議事録などのインプットデータを正確に見つけてくるのが難しい
- パソコンの中に散らばっていたり、もうどこにあるかわからない資料が大半
- アプローチ：成果物から逆算して「どのようなインプットからその成果物が得られそうか」をAIで推測させる
- 例：パワポのスライドがあるなら、それを作るために作ったであろう「テキストベースのスライドの骨子」をAIに生成させた
- この方法でエージェントを自動で学習させられる可能性
- リポスト元のスライド学習実験の引用あり

# 結果 / データ
- 実際に実験したところ、この方法はある程度有効であることがわかった（リポスト元の実験を参照）

# 考察 / 気づき
- 個人の使用用途に沿った暗黙知を吸収したエージェントを量産できる可能性
- データセット作成の難しさがSkills自動最適化のボトルネックだった
- 成果物からの逆算生成でそのボトルネックを解消できるかもしれない

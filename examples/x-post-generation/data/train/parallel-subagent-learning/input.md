これ# テーマ
スライド生成Skillsをサブエージェント並列化で学習・最適化する実験

# やったこと / 素材
- 過去のGoogleスライドを1枚ずつ分解してJSON化
- そこから骨子テキストを逆算してデータセットを作成
- 生成と評価を何イテレーションか回してスライド生成Skillsの改善を検証
- データセット自動生成の引用元投稿あり
- 1つの親エージェントで全部処理するのではなく、スライドのページごとにコンテキストを分離してサブエージェントに独立して生成させる
- 生成結果をまた別のサブエージェント群で並列評価する
- 例：20ページのスライドなら20個のサブエージェントで同時生成、評価も20ページ分を並列で走らせる
- 数十回、数百ケース、将来的には数百枚・数千枚規模で回していくことを想定
- コンテキスト分離の重要性：親エージェントに全てやらせるとコンテキストが漏れて正解データを見て生成してしまう可能性がある

# 結果 / データ
- train 56.7 → 73.3
- val 59.4 → 76.3
- スコアが改善した

# 考察 / 気づき
- スキルは、コンテキストを分離したサブエージェントを一度に大量に並列で動かしながら学習・最適化させるほうが本質的
- 答えが見れてしまうようなチートをせずに、タスクごとの独立性・コンテキストの分離を保ったまま学習ループを作れることがこのやり方の良さ
- 大量に並列で学習させる仕組みを作ることの方が重要
- 1回うまく作れたスキルより、データが増えるたびに更新され評価されまた改善されるスキルの方が価値が大きい
- そのための最小構成が少し見えてきた感覚
- 次は100枚、1000枚といった規模で回したときの振る舞いを見たい

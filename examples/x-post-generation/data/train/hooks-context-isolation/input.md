# テーマ
Skillsの最適化時にサブエージェントのコンテキスト分離をhooksで保証する仕組み

# やったこと / 素材
- 機械学習の基本原則：学習中にAIモデルが答えを絶対に見てはいけない。見ると学習が破綻する
- Claude Codeを用いたSkillsの最適化ではこの原則を守るのが少し難しい
- 親エージェントに「サブエージェントを用いてSkillsを実行して」と指示するだけでは、サブエージェントが間違って答えを見てしまう可能性がある
- 対策：Claude Codeの「hooks」機能を使用
- あるサブエージェントが `Read` しようとするたびに、必ず `./scripts/validate-forward-read.sh` が実行される
- スクリプト側で許可されたパスだけ読み込みを許可（例：`input.md` や特定のskillディレクトリ配下だけ）
- ルールベースで答えを絶対に見ないように制御する仕組み
- サブエージェント並列学習の引用元投稿あり

# 考察 / 気づき
- Skillsの最適化においてサブエージェントが独立したコンテキストで動いていることを保証する必要がある
- hooksを使ったルールベースの制御でコンテキスト漏洩を防止できる

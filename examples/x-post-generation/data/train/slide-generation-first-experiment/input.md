# テーマ
過去のスライドからテキスト骨子→スライド生成のSkillsを自動生成する最初の実験

# やったこと / 素材
- テキストの骨子を渡してスライドを生成するSkillsの自動生成を試みた
- 前回の提案書実験の引用元投稿がある
- データセットの作り方：
  - 手作業で過去に作ったGoogleスライドを一枚ずつに分離
  - オブジェクトが羅列されたJSONにする
  - 一枚ずつのスライドについて、Skillsの入力となるテキストでの骨子をAIで生成（リバースエンジニアリング的手法）
  - スライド7枚分を学習データ、3枚分をテストデータにする
- 学習プロセス：
  - テキストでのスライドの骨子を入力にしてSkillsを実行
  - スライドを生成（JSON形式）
  - 自動評価し、テキスト勾配を計算
  - Skillsを修正する、以降繰り返し
- Claude Codeを使って実験プロセス全体を自動化

# 結果 / データ
- 2ステップまではSkillsの中身も安定して改善
- 2ステップ以降は上がりづらくなった
- スライドのレイアウト、フォント、フォントサイズなどを厳密に評価できていなかった
- Skillsの更新の大きさが大きすぎるとスコアが悪化する

# 考察 / 気づき
- Skillsの成果物を評価するAIをタスクに応じて正しく厳密に定義することが大事
- 評価した結果を正しくSkillsにフィードバックする仕組みが大事
- 成果物を評価した後にそれをうまくSkillsに反映する仕組みを作ることが大切
- 実験自体の実行はそんなに大変ではないが、トークンを大量に消費する
- 最近発表されたAnthropicのskill-creatorにはSkillsを評価する仕組みもある（https://github.com/anthropics/claude-plugins-official/tree/main/plugins/skill-creator）

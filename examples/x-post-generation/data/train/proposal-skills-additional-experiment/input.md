# テーマ
提案書生成Skillsの自動最適化の追加実験結果

# やったこと / 素材
- 前回のSkills自動最適化プラグイン開発の続き（引用元の投稿がある）
- 実験条件：学習データ7件と評価データ3件に分離
- 学習データで学習させて、評価データは学習に使わずに評価のためだけに使用
- AI生成データをデータセットに使用した簡易的な実験

# 結果 / データ
- 約3イテレーションで過学習が発生する
- 1回ではなく3回くらいイテレーションを回すことで徐々にスコアが改善する
- 学習初期は「提案書に何を書くか」を学び、後期は「提案書のそれぞれの項目をどう書くか」を学ぶ
- 「全体最適→局所最適」の順番でAIが自動で学んだ

# 考察 / 気づき
- 全体最適→局所最適の順番で自動学習が進むのが面白い
- 今後は実際の過去の人の成果物を正解データとして使う方針
- DeepResearch、Notionツール、ほかMCPなどの使い方も自律的に学習させたい

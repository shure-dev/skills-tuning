# Xポスト生成の評価設定

## 評価方針

reference（実際に書いた投稿）と output（生成された投稿）を比較し、5軸の重み付きスコアで評価する。

## 評価軸（重み付き加重平均）

| 軸 | 重み | 評価観点 | スコア計算 |
|---|---|---|---|
| **content_coverage** | 15% | reference に含まれる事実・データ・固有名詞が output にも含まれているか | (output に存在する reference の情報断片数 / reference の情報断片総数) × 100 |
| **tone_match** | 25% | 語尾（敬体/常体）、口語度、一人称の使い方が reference と近いか | reference の語調特徴を抽出し、output の再現度を 0-100 で判定 |
| **formatting_match** | 20% | 箇条書き形式、段落の切り方、記号選択が reference と近いか | フォーマット要素の一致度を 0-100 で算出 |
| **flow_quality** | 25% | 導入→展開→締めの流れが自然か（reference との一致ではなく品質で評価） | 読み手にとっての論理展開の自然さを 0-100 で判定 |
| **conciseness** | 15% | 冗長な誘導語・不要情報がないか | 冗長表現の有無を 0-100 で判定（冗長さが少ないほど高スコア） |

## 各軸の詳細

### content_coverage（15%）

reference のテキストから情報断片（事実、数値、固有名詞、主張）を抽出し、output に含まれているか照合する。

- 同じ事実が言い換えられていても OK
- output に reference にない情報があってもペナルティなし
- reference の情報が output から欠落している場合に減点

### tone_match（25%）

reference の語調的特徴を抽出し、output が再現しているかを評価する。

- 敬体（です/ます）vs 常体（だ/である）の一致
- 口語的表現の度合い（「〜してみた」「〜だと思った」等）
- 一人称の使い方（「私は」「自分は」等の頻度・タイミング）
- 読者への語りかけ方（断定調 vs 共有調）

### formatting_match（20%）

reference と output のフォーマット要素を比較する。

- 箇条書きの使い方（番号付き vs 記号、入れ子の深さ）
- 段落の切り方（一文一段落 vs 複数文で一段落）
- 記号・装飾の選択（「」、（）、→、・ 等の使い方）
- 改行の入れ方のリズム

### flow_quality（25%）

output の文章としての流れの自然さを品質評価する。reference との構造一致ではなく、読み手にとっての品質で測る。

- 導入で読者の関心を引けているか
- 論理展開に飛躍がないか
- 結論・締めが唐突でないか
- 全体として一つのストーリーとして読めるか

### conciseness（15%）

output に冗長な表現や不要な情報がないかを評価する。

- 「まず」「次に」「そして」等の不要な接続詞の多用
- 同じ内容の繰り返し・言い換え
- input にない情報の不要な付け足し
- Xポストとして適切な簡潔さか
